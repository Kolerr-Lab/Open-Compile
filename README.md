## ‚ö° **Quick Start**

### Installation
```bash
npm install -g opencompile
# or
pnpm add -g opencompile
```

### Initialize Your Project
```bash
# Create from intent
opencompile create "REST API for user authentication with JWT"

# Detect existing project
opencompile detect

# Analyze and extend
opencompile analyze ./my-project
opencompile extend "Add GraphQL support"
```

### Configuration
```bash
# Set up AI providers (use .env file for security)
cp .env.example .env
# Edit .env and add your API key

# Or use config command
opencompile config set openai.api_key "your-key"
opencompile config set anthropic.api_key "your-key"

# Choose preferred model (default: gpt-4o)
opencompile config set ai.model "gpt-4o"
```

### ūüĒź **Security Best Practices**

**‚ö†ÔłŹ NEVER commit your `.env` file with real API keys!**

```bash
# ‚úÖ SAFE - Use .env file (already in .gitignore)
cp .env.example .env
echo "OPENAI_API_KEY=sk-your-real-key-here" >> .env

# ‚ĚĆ UNSAFE - Don't hardcode keys in code
const apiKey = "sk-your-key";  // NEVER DO THIS!

# ‚úÖ Verify before committing
git status  # Make sure .env is NOT listed
```

**What's Protected:**
- ‚úÖ `.env` is in `.gitignore` (never committed)
- ‚úÖ `.env.example` is committed (safe template)
- ‚úÖ Local `knowledge-base/` excluded (may contain project data)

**See [SECURITY.md](SECURITY.md) for complete security guidelines.**
